## 当前数据架构

```
┌─────────────────────────────────────────────┐
│             飞书多维表格 (Bitable)            │
│                                              │
│  tblEiMBFXcvSspQd  ← 招聘任务表              │
│  tblnT0AHtiLsvMeB  ← 年度招聘需求表           │
│  tblWkwsoTIPhzusI  ← 人才库表（小七写入）      │
│                                              │
│  缺少：简历表、面试表、日报表、岗位定义表...     │
└────────────────────┬────────────────────────┘
                     │  仅双向同步 (部分数据)
                     ▼
┌─────────────────────────────────────────────┐
│             Cloudflare D1 (主力库)           │
│                                              │
│  resumes          ← 简历（AI解析、初筛）       │
│  interviews       ← 面试（评分、评价）         │
│  talent_pool      ← 人才库                    │
│  job_requisitions ← 招聘需求                  │
│  daily_reports    ← 日报                     │
│  positions        ← 岗位定义                  │
│  users            ← 登录用户                  │
│  ...其他辅助表                                │
└──────────────────────────────────────────────┘
```

**问题**：D1 是主力数据库，飞书多维表格只存了部分数据（招聘任务和人才库），而且是"偶尔同步"的关系。简历、面试、日报这些只有 D1 有。取消面试时删除人才库记录，也只操作了 D1 和人才库表，面试记录本身在飞书没有对应。

## 要改为

```
┌─────────────────────────────────────────────┐
│        飞书多维表格 (Bitable) ← 数据主库      │
│                                              │
│  tblEiMBFXcvSspQd  ← 招聘任务                │
│  tblnT0AHtiLsvMeB  ← 年度招聘需求             │
│  tblWkwsoTIPhzusI  ← 人才库                   │
│  【新建】简历表     ← 简历 + AI解析结果         │
│  【新建】面试表     ← 面试记录 + 评分           │
│  【新建】日报表     ← 每日统计                  │
└────────────────────┬────────────────────────┘
                     │  直接读写 (API)
                     ▼
┌─────────────────────────────────────────────┐
│          Cloudflare D1 (本地缓存)            │
│                                              │
│  用来做：                                    │
│  1. 用户登录认证（密码不能放飞书）             │
│  2. 读缓存（减少飞书 API 调用）                │
│  3. 飞书 API 不可用时的降级                   │
└──────────────────────────────────────────────┘
```

这样所有业务数据（简历、面试、人才库）都以飞书多维表格为准，取消面试时直接在飞书更新面试状态 + 删人才库记录，一步到位。
