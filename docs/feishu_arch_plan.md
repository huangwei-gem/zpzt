## 改造计划

### 目标
所有业务数据直接读写飞书多维表格，不经过 D1。D1 仅保留用户认证和面试记录（用户说面试先不管）。

### 涉及的表
| 飞书表 | 系统模块 | 操作 |
|--------|---------|------|
| `tblEiMBFXcvSspQd` 招聘任务 | 需求管理 | 直接读/写 |
| `tblWkwsoTIPhzusI` 人才库 | 人才库、简历管理/初筛 | 直接读/写 |
| D1 原有表 | users | 保留（密码不能放飞书） |
| | interviews | 暂时保留 |
| | daily_reports | 暂时保留 |

### 改造步骤
1. 后端：写通用 Bitable CRUD 辅助函数（list/get/create/update/delete）
2. 后端：替换需求管理的 CRUD → 用招聘任务表
3. 后端：替换人才库 CRUD → 用人才库表
4. 后端：替换简历管理/初筛 CRUD → 用人才库表（候选人和 AI 分析都在里面）
5. 前端：移除同步按钮（数据是实时的），适配新字段名
6. 测试
